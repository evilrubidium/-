# Задача 2 
1. высоконагруженная база данных MySql, критичная к отказу - тут я выберу физические сервера, потому что Максимально прямой доступ к дисковой подсистеме, поэтому мы будем экономить время при работе с высоконагруженной БД. Можно также паравиртуализацией воспользоваться потому что удобно реализовать отказоустойчивость и современные гипервизоры дают минимальные потери производительности (5–10%)
2. различные web-приложения - здесь я выберу виртуализация уровня ОС (контейнеры), тк контейнеры будут быстро запускаться и будут очень низкие накладные расходы, сможем неплохо преуспеть в расвертывании и неплохо съекономить на этом
3. Windows-системы для использования бухгалтерским отделом - паравиртуализацию выберу, мы сможем изолировать пользователей, для разграничения доступа; при необнодимости сможешь делать снапшоты версий и копировать ВМ. Если системы откажут, сможем быстро собрать и восстановить
4. Системы, выполняющие высокопроизводительные расчёты на GPU - только на физических серверах, у нас будет полный доступ ко всей мощности GPU, съекономим на отсуствии виртуализации, и получим быстро результат изза мощности как раз таки GPU

# Задача 3
100 виртуальных машин на базе Linux и Windows, общие задачи, нет особых требований. Преимущественно Windows based-инфраструктура, требуется реализация программных балансировщиков нагрузки, репликации данных и автоматизированного механизма создания резервных копий. - здесь бы я скорее предпочел воспользоваться Microsoft Hyper-V, он сможет запустить как Windows так и Linux системы, + неплохо будет справляться с бекапами
Требуется наиболее производительное бесплатное open source-решение для виртуализации небольшой (20-30 серверов) инфраструктуры на базе Linux и Windows виртуальных машин. - в данном случае я попробовал бы KVM, как раз будет подходить под запрос производительности виртуализации.
Необходимо бесплатное, максимально совместимое и производительное решение для виртуализации Windows-инфраструктуры. - тут опять же прибегнул бы к Microsoft Hyper-V, меня устраивает наличие свежих драйверов и наличие широкого гостевого инструментала
Необходимо рабочее окружение для тестирования программного продукта на нескольких дистрибутивах Linux. - тут уже KVM + virt-manager. Это комбинация хорошо сработает, тк у нас ограничение по системе Linux стоит,  с теми задачами с которыми отлично решит

# Задача 4
Во-первых, у каждом системы свои параметры конфигурации и других настроек, т.е. мы уже изначально будет тратить время и ресурсы чтобы синхронизировать как минимум работу систем. Во-вторых, каждая система по своему делает бекапы и снапшоты, версии будут отличатся и моментами будет тяжело воедино потом собирать всё до кучи. В-третьих проблема совместимости версий, мы можем столкнутся с проблемой миграции с одной системы к другой.
